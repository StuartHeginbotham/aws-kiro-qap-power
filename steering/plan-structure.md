# Quick Action Plan — Output Structure

The final QAP document should follow this structure exactly.

---

```markdown

**Created:** [DATE]
**Owner:** [NAME]
**Status:** Draft
```

## Milestone Field Definitions

| Field | Description |
|-------|-------------|
| **Name** | Short, clear label (3-5 words) |
| **Target Date** | When this milestone should be complete |
| **Effort** | Rough size — use the user's language (days, weeks, sprints, story points) |
| **Outcome** | What's delivered or resolved. Must be independently valuable. |
| **Success Criteria** | How you know it's done. Measurable where possible. Checkboxes. |
| **Key Actions** | The 3-5 things that need to happen. Ordered by sequence. |
| **Dependencies** | What must be true before this milestone can start. Include prior milestones. |

## Rules for Good Milestones
1. Each milestone delivers value on its own — if the plan stops here, something useful was achieved
2. Milestones are roughly equal effort (±20%)
3. Milestone 1 targets the most urgent pain or highest-value quick win
4. The final milestone represents the "awesome" success state
5. No milestone should have more than 5 key actions — if it does, split it
